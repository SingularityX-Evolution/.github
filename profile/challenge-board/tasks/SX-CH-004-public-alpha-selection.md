## 背景

在 AI 量化项目中，可以快速复现大量公开经典策略，例如均线交叉、Donchian 突破、RSI / Bollinger 均值回归、时间序列动量、简单配对交易、波动率目标仓位等。问题不在于复现这些策略，而在于如何通过数据挖掘和严谨样本外验证，从公开策略池中筛选出扣除成本后仍具备稳定收益和高 Sharpe 的组合。

该问题具有实际难度：公开策略参数空间大，容易出现训练集高收益、样本外失效；金融市场存在非平稳和 regime change；策略收益还会被手续费、滑点、资金费率和换手成本侵蚀。单纯让大模型生成策略说明或简单网格搜索，通常难以稳定通过严格的样本外验证。

## 核心问题

如何基于公开经典量化策略和公开市场数据，构建一个可运行的数据挖掘与策略筛选模块，自动发现稳定收益、高 Sharpe、低回撤、扣除真实交易成本后仍然有效的策略组合，并在样本外数据中证明其收益不是由过拟合、未来函数或偶然行情造成？

## 问题范围

本题聚焦公开经典策略池的 Alpha 挖掘、筛选、组合与样本外验证。参赛者不需要构建完整交易系统，不需要接入真实交易所，不需要做订单执行算法，也不得使用任何主办方内部信号或盈利逻辑。

## 公开输入与输出要求

参赛者需要实现一个模块：

```text
M(B, D, C, R) → P
```

其中，`B` 为公开策略池，`D` 为公开市场数据，`C` 为交易成本配置，`R` 为风险约束，`P` 为最终挖掘出的策略组合、权重、参数和启停规则。

| 类别 | 示例字段 | 要求 |
|---|---|---|
| 公开策略池 | `strategy_id`、`family`、`parameters` | 仅限公开经典策略；不得包含内部经验参数 |
| 市场数据 | `timestamp`、`symbol`、`OHLCV`、`spread`、`funding`、`open_interest` | 资产可脱敏为 `ASSET_001` 等；严格时间戳对齐 |
| 成本配置 | maker / taker fee、slippage、funding、tick / lot / min order | 所有收益应扣除成本 |
| 输出组合 | `selected_strategies`、`parameter_set`、`weight`、`enabled_regime` | 应可复现、可回测 |
| 逐期持仓 | `timestamp`、`symbol`、`target_position`、`active_strategy_id` | 用于复现与结果核查 |

## 交付标准

有效提交应至少包含以下内容：

1. **完整工程仓库**
   - README.md、Dockerfile 或等价环境配置、依赖文件、配置文件和测试脚本；
   - 主办方在隔离环境中可一键复现；
   - 能生成最终组合、参数、权重、逐期持仓、trial 搜索记录和评估报告。

2. **公开策略池与搜索记录**
   - 策略必须来自公开经典策略池，明确策略来源、参数空间和启停规则；
   - 输出完整 trial 记录，包含每次搜索参数、训练期表现、验证期表现、是否入选和淘汰原因；
   - 不得使用内部信号、未披露经验参数或隐藏人工筛选。

3. **成本、风险与组合约束**
   - 所有收益必须扣除交易成本，包括 fee、slippage、funding 和换手成本；
   - 输出逐期持仓、仓位变化、换手、成交假设和风险约束；
   - 明确最大仓位、杠杆、单标的暴露、回撤控制和停用规则。

4. **隐藏样本外评测准备**
   - 模块应能接受新的市场数据、成本配置和风险约束，并重新生成组合与评估报告；
   - 不得依赖训练集固定日期、固定资产名称或人工后验挑选。

5. **反作弊与反过拟合报告**
   - 不使用 `future_return`、`close_t_plus_1`、`next_bar_high/low` 等未来字段；
   - 提供至少 3 项有效反过拟合报告，包括但不限于 PBO、DSR、walk-forward、purged split、CPCV、label permutation、synthetic null；
   - 提供 baseline 对比，包括 Buy and Hold、No Trade、公开策略等权组合、公开策略风险平价组合、Random Strategy with Same Turnover、训练期 Sharpe 最高的朴素选择法。

## “完全解出”的明确标准

获得全额 12,000 元，必须同时满足以下条件：

1. **工程可运行**
   - 提交完整代码仓库；
   - 包含 README.md、Dockerfile 或等价环境配置、依赖文件、配置文件、测试脚本；
   - 主办方在隔离环境中可一键复现；
   - 能生成最终组合、参数、权重、逐期持仓、trial 搜索记录和评估报告。

2. **隐藏样本外表现达到优秀线**
   - 年化 Sharpe ≥ 1.8；
   - 扣成本后净收益 > 8%；
   - 最大回撤 ≤ 3%；
   - Calmar Ratio ≥ 2.0；
   - 月度正收益比例 ≥ 65%；
   - 单一最好月份收益占总收益比例 ≤ 30%。

3. **显著优于强制基线**
   - 优于 Buy and Hold；
   - 优于 No Trade；
   - 优于公开策略等权组合；
   - 优于公开策略风险平价组合；
   - 优于 Random Strategy with Same Turnover；
   - 优于训练期 Sharpe 最高的朴素选择法。

4. **通过反作弊与反过拟合测试**
   - 不使用 `future_return`、`close_t_plus_1`、`next_bar_high/low` 等未来字段；
   - synthetic null market 中不得继续报告异常高 Sharpe；
   - label permutation 后不得继续表现稳定盈利；
   - cost shock 后策略不得明显崩溃；
   - 必须提供至少 3 项有效反过拟合报告，包括但不限于 PBO、DSR、walk-forward、purged split、CPCV、label permutation、synthetic null。

5. **可复现与可审计**
   - 提交的每个入选策略、参数、权重和启停规则均可追溯到 trial 记录；
   - 逐期持仓与回测结果可由代码重新生成；
   - 任何人工过滤、数据清洗或异常处理必须记录依据；
   - 不接受只提交最终收益曲线或无法复核的组合结果。

## 无效提交认定

以下情况可判定为无效提交或未通过等级认证：

- 无法运行、无法复现或缺少核心代码；
- 仅提交文字方案、PPT、截图或无法验证的收益曲线；
- 无 Docker 或等价环境配置，导致主办方无法复现；
- 无逐期持仓、无 trial 搜索记录、无参数和权重记录；
- 未扣除交易成本，或成本扣除口径不清；
- 使用未来字段、未来收益、未来高低点或 hindsight 筛选；
- 未提供强制 baseline 对比；
- 反作弊测试、label permutation、synthetic null 或 cost shock 失败严重；
- 使用主办方内部信号、私有盈利逻辑或未披露人工筛选。
