## 背景

真实的因果链是多跳的，例如：

```text
中东冲突 → 霍尔木兹海峡封锁风险 → 原油供应紧张 → 油价上涨 → 化工原料成本上升 → 化工板块做空
```

每一跳都会引入额外的不确定性，且跳数越多，干扰因素越多。例如第 3 跳到第 4 跳之间可能已经出现 OPEC 增产来对冲前序影响。

给出一个最初的中东冲突关键词后，系统希望在不借助 LLM 的情况下，通过构建数据库查阅或自训练模型快速向后推理跳跃，最终推理到交易相关信号，并保证推理链条流畅、通顺。

## 核心问题

目前没有成熟的方法自动计算“这条 3 跳链的综合置信度是多少”。简单连乘，例如每跳 0.7、三跳后得到 0.34，在理论上合理但实践中太保守；不做衰减则会高估长链可靠性。

更根本的问题是：什么时候应该停止跳跃？多跳意味着推理更完整，但也意味着噪声更多。该挑战关注的是如何为多跳因果链建立可解释、可校准的不确定性度量，并形成具有原则性的停止准则。

## 公开任务范围

参赛者需要围绕多跳因果链的不确定性量化与停止准则提出可落地方法。方案可以包括因果链表示、每一跳置信度建模、跨跳不确定性传播、噪声控制、停止准则设计，以及与交易相关信号衔接的系统化思路。

本题不要求构建完整因果知识图谱；链结构和每跳初始置信度可以由调用方给定。核心任务是解决**非朴素连乘的置信度聚合、外部干扰建模、端到端校准与停止准则**。

## 交付标准

有效提交应至少包含以下内容：

1. **方法文档**
   - 说明朴素连乘失效的数学原因，包括独立性假设、相关性、外部截断事件、长链衰减过强或过弱等问题。
   - 给出至少一种非朴素连乘聚合方案，例如对数几率聚合、贝叶斯逐跳更新、干扰衰减模型、Noisy-OR / 多路径聚合或等价模型。
   - 明确每个变量的含义、取值范围、可解释性和边界条件。

2. **停止准则**
   - 必须给出原则性停止规则，例如置信区间宽度阈值、剩余置信度楼层值、期望信息增益阈值、最大跳数约束或组合规则。
   - 需说明停止准则为何能避免“无限跳跃”和“过早停止”。

3. **可运行实现**
   - 提交完整代码仓库或结构化代码包。
   - 至少包含 README.md、依赖文件、配置样例、运行脚本和最小测试用例。
   - 输入应支持链结构、每跳初始置信度、可选干扰项或路径元数据；输出应包含端到端置信度、置信区间、是否停止、触发停止原因和中间计算记录。

4. **校准验证**
   - 提供历史标注链或合成验证链的构造方法。
   - 尽量在不少于 50 条标注历史链上评估；如果无法构造足量历史数据，必须给出可复核的数据构造方案和局限说明。
   - 至少报告 Brier Score、可靠性图或分桶校准表、不同跳数实际胜率趋势。

5. **评估报告**
   - 对比朴素连乘 baseline 与提交方案。
   - 说明参数估计方式、敏感性、失败案例和适用范围。
   - 对“天然不可校准”或“数据不足”的结论给出证据，不接受仅凭主观判断。

## “完全解出”的明确标准

获得全额 7,200 元，必须同时满足以下条件：

1. **工程可运行**
   - 提交完整代码仓库；
   - 包含 README.md、依赖文件、配置样例、运行脚本和测试脚本；
   - 主办方在隔离环境中可复现主要结果；
   - 能输出端到端置信度、每跳中间量、置信区间、停止判断和评估报告。

2. **非朴素聚合模型成立**
   - 至少实现一种理论合理的非朴素连乘方案；
   - 明确区分链路自身断裂与外部事件截断；
   - 说明相关性、先验、干扰项或多路径聚合的处理方式；
   - 与朴素连乘 baseline 做定量对比。

3. **停止准则可解释且可执行**
   - 停止规则必须能在代码中触发并输出原因；
   - 需支持置信区间宽度、信息增益、置信度下限或最大跳数等至少一种原则性标准；
   - 不能仅用固定跳数作为唯一解释。

4. **校准验证充分**
   - 在不少于 50 条标注历史链上完成校准验证，或提供经主办方认可的等价验证集构造方法；
   - Brier Score < 0.15；
   - 提供可靠性图或等价分桶校准表；
   - 展示不同跳数的实际胜率衰减趋势。

5. **报告完整**
   - 报告需覆盖公式推导、参数估计、实验设置、baseline、结果、误差来源、局限性和可扩展方向；
   - 对失败案例和边界条件给出解释。

## 无效提交认定

以下情况可判定为无效提交或不通过：

- 仅提交概念性文字、PPT、截图或无法验证的结论；
- 未给出可复核的聚合公式或停止准则；
- 只使用朴素连乘且无替代方案；
- 输出结果无法解释每跳贡献、外部截断或不确定性传播；
- 无法运行、缺少核心代码或依赖无法安装；
- 使用结果标签、未来事件或人工 hindsight 调整置信度却未披露；
- 无 baseline 对比、无验证方法、无误差分析；
- 伪造验证数据或无法提供数据构造依据。
