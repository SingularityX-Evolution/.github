每位提交者对同一挑战仅有 1 次首次提交机会。仅当首次提交获得 A 档认证时，可自 A 档认证确认邮件发出之时起 5 个自然日（120 小时）内，再提交 1 个仅用于申请 S 档认证的完整升级版本；每人每题合计最多 2 次提交，逾期视为放弃升级机会。升级提交达到 S 档标准的，最终按 S 档认证和结算；A 档奖金已支付的，仅补发差额，不重复领取两档奖金；未达到 S 档的，保留原 A 档认证且不重复发放 A 档奖金。每次提交均以完整材料送达主办方指定邮箱或指定上传渠道的时间戳为准进入评审队列。某档位认证成功并由主办方公告关闭后，不再接受该档位新的认定申请；A→S 升级机会不重新开放已经关闭的 S 档，仅可在升级提交送达时 S 档仍开放的情况下使用。关闭公告前已经收到的提交，主办方仍会给出答复，但不保证该档位仍有奖金名额或继续进入该档奖金评审。

## 难度与赏金说明

难度与人民币赏金上限按完整挑战维度设置，不是子任务报价、阶段付款或保证付款。实际赏金认定以提交材料、复现结果、评审结论、贡献证据、适用协议、合规要求和赏金评审流程为准。

## 提交方式

私密提交发送至：

```text
join@singularityx.tech
```

建议邮件标题：

```text
[Challenge Submission] <Challenge ID> - <Contributor Name or GitHub Username>
```

有效提交应包含可复现代码或结构化交付包、README 与运行说明、测试结果或评估报告、依赖与环境信息、已知限制，以及贡献者确认其拥有或有权提交相关成果的声明。

首次提交和 A→S 升级提交均视为独立版本，完整材料送达后不得修改；升级提交须提供新的完整版本并单独计次。提交或上传后五个工作日内，主办方会给出答复。答复可能包括：确认进入评审、要求补充非实质性材料、初筛未通过、继续复现评测、通过某档认证或未通过等级认证。

等级认证成功后，主办方仅会在不披露敏感信息、私有代码、账户信息、未公开策略细节和受限数据的前提下，公示认证成功者的脱敏作品摘要、认证档位、挑战编号或经审核可公开材料，公示期为一周。
